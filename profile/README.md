## 第一步：确认服务器是否正常运行

这步很多人直接跳过，结果找了半天问题，才发现是服务器自己关机了。

登录搬瓦工账号，进入 **Services → My Services**，找到你的 VPS 实例，点击进入 **KiwiVM Control Panel**。

## 第二步：检测 IP 是否被封

这是搬瓦工 IP 被封检测最关键的一步，有两种方法，建议都用一遍互相印证。

### 方法一：搬瓦工官方 KiwiVM 检测工具（最权威）

这是搬瓦工自己做的工具，检测结果最准。

**操作步骤：**

1. 登录 KiwiVM 控制面板（在搬瓦工后台点进你的 VPS 实例）；
2. 在浏览器地址栏，把 URL 中的 `main-exec.php` 后面改成 `?mode=blacklistcheck`；
   - 完整链接格式：`https://kiwivm.64clouds.com/你的VPS-ID/main-exec.php?mode=blacklistcheck`
3. 进入页面后，点击 **Test Main IP** 按钮；
4. 等待几秒，看结果。

**两种结果对应含义：**

| 检测结果 | 含义 |
|---------|------|
| **IP NOT BLOCKED** | IP 正常，没有被封，问题在别处 |
| **IP BLOCKED** | IP 已被封，国内无法访问 |

> 需要注意：官方工具在中国高峰时段（晚上 8–11 点）可能不太准，如果结果有疑问，配合方法二再验证一次。

### 方法二：第三方工具 ping.pe（快速直观）

ping.pe 是目前口碑最好的 VPS IP 可用性检测工具之一，可以从全球多个节点同时 ping 你的 IP，直观看到国内外的连通情况。

**操作步骤：**

1. 打开 [ping.pe](https://ping.pe)；
2. 在输入框里填入你的搬瓦工 IP 地址；
3. 点击测试，等结果出来。

**如何判断结果：**

- **国外节点全绿、国内节点全红**：IP 被封了，国内无法访问；
- **国内外都绿**：IP 正常，问题出在别处；
- **国内外都红**：服务器本身有问题，或机房故障。

## 第三步：如果 IP 没被封，检测端口是否被封

很多人遇到的情况是：IP 能 ping 通，但 SSH 就是连不上。这种情况大概率是 **SSH 端口被封**了，而不是 IP 问题。

IP 被封是整个 IP 地址都不通，端口被封是 IP 能 ping 通但特定端口走不了。这是两个不同层面的问题，解决方法也不一样。

### 端口检测方法：port.ping.pe

1. 打开 [port.ping.pe](https://port.ping.pe)；
2. 输入格式：`你的IP:端口号`（例如 `1.2.3.4:20000`）；
3. 点击测试。

**判断结果：**

- **境外 successful + 境内 failed**：端口被封了；
- **境内外都 successful**：端口没问题，检查下你本地的网络或防火墙；
- **境内外都 failed**：端口没开放，检查服务器防火墙设置。

**如果端口被封，怎么办：**

在 KiwiVM 控制面板 → **Root Shell** 里，或者通过 VPS 的 Web 控制台，修改 SSH 端口到一个新的随机端口（建议 10000–65000 之间），然后重启 SSH 服务即可。

## 搬瓦工 IP 为什么会被封？

了解封禁原因，才能从根本上避免这个问题反复发生。

主要原因集中在以下几类：

**1. 使用方式触发封禁**：在国内被判定为违规用途的流量行为，会导致 IP 被 GFW 拉黑。

**2. 套餐线路质量**：普通 KVM 或 CN2 GT 线路使用的 IP 段相对"嘈杂"，比 CN2 GIA 专线更容易被封。CN2 GIA 线路由于带宽价格更贵，运营者更在意 IP 口碑，整体稳定性好很多。

**3. 共享 IP 段被连带**：你的 IP 本身没问题，但和你同网段的 IP 因为其他用户的行为被封，导致你被连带。这种情况换 IP 后一般就解决了。

**4. 机房或时段问题**：某些机房在特定时段（比如敏感时间节点前后）会有集中封禁，通常持续一段时间后会恢复。

## 常见问题排查与错误处理

**Q：官方检测工具打不开怎么办？**

先确认你的 KiwiVM 面板是否已登录，链接中的 VPS-ID 是否正确。也可以直接用 ping.pe 替代。

**Q：ping.pe 检测结果是"国内大部分节点 failed，但有一两个 successful"，算被封吗？**

如果只是个别节点 offline（测试节点本身挂了），不影响判断。如果是大陆主要运营商（电信、联通、移动）的节点都是 failed，基本可以判断 IP 被封。

**Q：IP 被封了，能不能通过迁机房换 IP 来解决？**

不行。搬瓦工的规则是：IP 被封时无法迁机房，必须先换 IP（$8.79）或者等 IP 解封，之后才能迁移。

**Q：换了新 IP 之后，还是被封怎么办？**

这种情况说明使用行为本身存在问题，建议重新审视 VPS 的使用方式。另外也可以考虑换 CN2 GIA 线路的套餐，IP 稳定性从根本上提升。👉 [查看 CN2 GIA-E 套餐](https://bwh81.net/aff.php?aff=80238&pid=87)

**Q：搬瓦工 IP 被封和端口被封，有什么区别？**

- **IP 被封**：这个 IP 地址整体被 GFW 屏蔽，不管什么端口都不通，ping 也 ping 不通；
- **端口被封**：IP 正常，但某个特定端口（比如 SSH 的 22 或你设置的端口）被拦截，表现是 ping 通但 SSH 无法连接。解决方法不同：IP 被封要换 IP，端口被封换个端口就好。
